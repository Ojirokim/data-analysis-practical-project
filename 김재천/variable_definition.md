# [교육] 온라인 강의 사용자 로그 분석 및 시각화 - 변인 정의서

## 1. 개요 및 배경
* 온라인 강의 플랫폼에서의 **학생 참여도(Student Engagement)**는 오프라인 교실과 달리 학생의 집중도나 반응을 직접 관찰할 수 없기 때문에 **"학습 행동의 흔적(로그 데이터)"**으로 측정됩니다.
* 본 데이터는 학습자의 등록부터 수료까지의 전반적인 여정을 추적하고, 강의 콘텐츠와 플랫폼의 개선점을 찾는 데 핵심적으로 활용됩니다.

## 2. 주요 퍼널(Funnel) 정의
단일 변수뿐 아니라 여러 변수를 결합해 학습자의 행동을 하나의 흐름(Funnel)으로 정의할 수 있습니다.
* **1단계 (등록)**: `registered`
* **2단계 (첫 방문)**: `viewed`
* **3단계 (적극적 탐색)**: `explored`, `nchapters`, `nplay_video`
* **4단계 (지속 학습 및 상호작용)**: `nevents`, `ndays_act`, `nforum_posts`
* **최종 단계 (인증/수료)**: `certified`

## 3. 컬럼별 변인 정의서 (Codebook)

| 변수명 (Column) | 데이터 타입 | 설명 (Description) | 주요 의미 및 활용안 |
|---|---|---|---|
| `index` | int64 | 행 인덱스 | 데이터 정렬 및 고유 행 식별용 |
| `course_id` | object | 강의 ID | 강의별 퍼널 전환율 및 성과 비교 |
| `userid_DI` | object | 학습자 식별자 | 개별 유저의 행동 추적 및 코호트 매핑 |
| `registered` | int64 | 등록 여부 | 강의에 수강 신청을 했는지 (1=등록) |
| `viewed` | int64 | 강의 열람 여부 | 수강 신청 후 실제 강의 페이지를 1회 이상 열어봤는지 |
| `explored` | int64 | 콘텐츠 적극 탐색 여부 | 강의 콘텐츠를 단순 클릭에 그치지 않고 깊게 탐색했는지 |
| `certified` | int64 | 인증서 획득 여부 | 최종 수료 여부 (학습 여정의 최종 목적지) |
| `final_cc_cname_DI` | object | 국가 (Country) | 국가별 선호 강의 파악 및 리전(Region)별 참여율 분석 |
| `LoE_DI` | object | 최고 학력 수준 (Level of Education) | 학력 분포 파악 및 학력 수준이 완주율에 미치는 영향 |
| `YoB` | float64 | 출생연도 (Year of Birth) | 수강 시작일(`start_time_DI`)과 결합해 연령(Age) 파생 변수 도출 |
| `gender` | object | 성별 | 남/녀/기타 집단 간의 학습 활동 양상 비교 |
| `grade` | object | 학습 성취 수준 (성적) | 평가 점수 (숫자 타입 변환 필요, 0점으로 결측치 대치) |
| `start_time_DI` | object | 강의 시작일 (세션 시작일) | 유저 코호트 집단 생성 및 경과 수강 일수 계산 기준점 |
| `last_event_DI` | object | 마지막 활동일 (마지막 로그 발생일) | `start_time_DI`와의 차이로 전체 학습 기간(`duration_days`) 등 도출, 이탈 시점 파악 |
| `nevents` | float64 | 전체 학습 이벤트 수 | 전반적인 플랫폼 상호작용 지표 (페이지 방문, 클릭 버튼 수 등) |
| `ndays_act` | float64 | 실제 활동한 일수 | 얼마나 꾸준히 학습 플랫폼에 접속했는지 (학습 지속성) |
| `nplay_video` | float64 | 영상 재생 수 | 시청각 매체에 대한 관여도 및 강의 난이도(다시보기 여부) 간접 지표 |
| `nchapters` | float64 | 탐색한 챕터 수 | 얼마나 다양한 챕터(커리큘럼)를 넘나들며 시청했는지의 지표 |
| `nforum_posts` | int64 | 포럼 게시글 수 | 교수자 및 다른 학습자와의 소셜/커뮤니티 상호작용 수준 |
| `roles` | float64 | 사용자 역할 | 시스템 내 유저 권한. 대부분의 행이 결측치이므로 제외 고려 |
| `incomplete_flag` | float64 | 불완전 데이터 관련 플래그 | 결측치가 다수 포함되어 있어 제외 또는 플래그성 변수로만 취급 고려 |

## 4. 이해관계자 (Stakeholders) 및 데이터 활용 아이디어

* **교육운영팀**
  * **Pain Point 파악**: `registered` 수는 높으나 `viewed`나 `explored` 비율이 현저히 낮은 코스, 즉 "수업을 등록만 해두고 안 듣는" 허수 수강생이 많은 코스를 발굴합니다.
* **콘텐츠 기획자 / 강사**
  * **수업 리뉴얼 인사이트**: `grade`(성적)와 `nplay_video`(영상 재생 빈도), `nchapters`를 교차 조회하여 유독 특정 강좌에서 영상 복습이 잦거나 진도율이 막히는 구간의 난이도를 조정합니다.  
* **제품팀 / PO (Product Owner)**
  * **기능 고도화**: `ndays_act`가 높은 활성 사용자들의 행태 패턴(`nforum_posts` 등)을 분석해, 학습 중간에 흥미를 잃어버리는(`last_event_DI`가 오래된) 유저에게 푸시 알림이나 커뮤니티 참여 유도를 설계합니다.
* **사업팀 / 임원**
  * **성과 평가**: 최종 `certified`(인증) 비율이 높은 강의나 `LoE_DI`/`final_cc_cname_DI` 등 타겟 고객군에 크게 어필하는 특정 강좌를 판별해 향후 마케팅 비용 배분과 신규 사업 기획에 반영합니다.
